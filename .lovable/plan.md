

# תיקון Hero Section + בדיקת ניהול תוכן

## מצב נוכחי

### Hero Section
כרגע ה-Hero משתמש ב-`min-h-[350px]` במובייל ו-`min-h-[600px]` בדסקטופ, עם `object-cover` שחותך את התמונה. אין יחס קבוע — הגובה קבוע והתמונה נחתכת בצורה שונה בכל מסך.

### איפה מעדכנים את תמונת ההירו?
**ניהול אתר → דף ראשי → באנר ראשי → תמונת רקע** — זה כבר עובד ומחובר ל-DB (בסיס הנתונים כבר מכיל תמונה מעודכנת).

### מה מנוהל ב-CMS ומה לא?
**מנוהל בבסיס נתונים (ניהול אתר):**
- דף ראשי: באנר ראשי (כותרת, תת-כותרת, כפתור, תמונת רקע)
- דף אודות: באנר, סיפור, ערכים
- דף צור קשר: פרטי התקשרות
- הגדרות כלליות (לוגו, טלפון, רשתות חברתיות)

**לא מנוהל — תמונות hardcoded מ-assets:**
- תמונות קטגוריות בדף הבית ובדף החנות (כבר הועברו ל-DB אבל עדיין יש fallback ל-assets)
- תמונות דף אודות: hero, story, 3 ערכים (aboutHero, aboutStory, aboutValue1-3) — **לא מנוהלות ב-CMS**
- לוגו בצ'קאאוט

## פתרון מוצע — Hero Aspect Ratio

במקום `min-h` קבוע, נשתמש ב-**aspect ratio קבוע** שמתאים גם למובייל וגם לדסקטופ.

יחס מומלץ: **16:9** — מתאים טוב למובייל (לא גבוה מדי) וגם לדסקטופ (נראה סינמטי). תמונה אחת תעבוד בשניהם.

### שינוי ב-`WebHome.tsx`:
- הסרת `min-h-[350px] md:min-h-[600px]`
- שימוש ב-`aspect-[16/9]` על הסקשן
- התמונה תוצג במלואה עם `object-cover` (חיתוך מינימלי ביחס אחיד)

### קובץ: `src/pages/web/WebHome.tsx`
שורה 44 — שינוי class של הסקשן מ:
```
relative overflow-hidden min-h-[350px] md:min-h-[600px] flex items-center
```
ל:
```
relative overflow-hidden aspect-[16/9] max-h-[600px] flex items-center
```

זה מבטיח:
- יחס זהה בכל מסך
- במובייל: ~210px גובה על מסך 375px רוחב (אפשר לשנות ליחס אחר אם רוצים יותר גובה)
- בדסקטופ: מוגבל ל-600px מקסימום

> **שאלה:** 16:9 עלול להיות נמוך מדי במובייל. אולי עדיף **3:2** שנותן יותר גובה במובייל? או **2:1**?

## לגבי תוכן לא מנוהל

תמונות דף ה-About (hero, story, values) הן hardcoded ולא ניתנות לעדכון מה-CMS. אם תרצה, אפשר להוסיף אותן גם לניהול התוכן בשלב הבא.
