

# שיפור UX/UI — צמצום דפים ואיחוד מסכים

## רעיון מרכזי
הרבה דפים במערכת הם "רשימות פשוטות" שלא מצדיקות דף נפרד. נאחד אותם לתוך דפים הגיוניים כ-Dialogs/Tabs.

## שינויים מוצעים

### 1. קטגוריות → פופאפ בדף הפריטים
- הסרת `/inventory/categories` כדף נפרד
- הוספת כפתור "ניהול קטגוריות" בדף `ProductsPage.tsx` שפותח Dialog עם טבלת קטגוריות + CRUD
- הסרת "קטגוריות" מהתפריט הצדי

### 2. מחסנים → פופאפ בדף תצוגת מלאי
- הסרת `/inventory/warehouses` כדף נפרד
- הוספת כפתור "ניהול מחסנים" בדף `InventoryIndex.tsx` שפותח Dialog
- הסרת "מחסנים" מהתפריט

### 3. לוג תנועות → כבר קיים בדוחות
- לוג התנועות כבר מוטמע בדף הדוחות (לשונית "תנועות מלאי")
- הסרת `/inventory/log` כדף נפרד מהתפריט (Route נשאר לגישה ישירה)

### 4. קופות → הגדרות (ולא פריט נפרד בתפריט ראשי)
- הסרת "קופות" מהתפריט הראשי — הגישה רק דרך דף הגדרות (כבר מקושר שם)
- קופות הן דף ניהולי, לא תפעולי יומיומי

### 5. איחוד הוצאות + מסמכים → דף אחד עם Tabs
- איחוד "הוצאות" ו"מסמכים" לדף אחד "פיננסי" עם 2 לשוניות
- הסרת שני פריטים מהתפריט, הוספת פריט אחד "כספים"

### 6. היסטוריית קליטות → Tab בדף הקליטה
- במקום דף נפרד, הוספת Tab "היסטוריה" בתוך `IntakePage.tsx`
- הסרת "היסטוריית קליטות" מהתפריט

## תפריט צד חדש (מ-15 פריטים ל-~9)

```text
דשבורד
מלאי
  ├── תצוגת מלאי  (+ כפתור ניהול מחסנים)
  ├── פריטים       (+ כפתור ניהול קטגוריות)
  ├── מארזים
  ├── קליטת מלאי   (+ Tab היסטוריה)
  └── העברות
פריטי אתר
הזמנות
משלוחים
קופה (POS)
כספים (הוצאות + מסמכים)
דוחות (כולל לוג תנועות)
הגדרות (כולל קופות, חברות משלוח, WooCommerce)
```

## קבצים לשינוי

| פעולה | קובץ | תיאור |
|---|---|---|
| שינוי | `ProductsPage.tsx` | הוספת Dialog לניהול קטגוריות |
| שינוי | `InventoryIndex.tsx` | הוספת Dialog לניהול מחסנים |
| שינוי | `IntakePage.tsx` | הוספת Tab היסטוריה |
| חדש | `FinancePage.tsx` | איחוד הוצאות + מסמכים עם Tabs |
| שינוי | `AppSidebar.tsx` | צמצום תפריט |
| שינוי | `App.tsx` | עדכון routes, הוספת `/finance`, הסרת routes מיותרים |
| שינוי | `SettingsPage.tsx` | הוספת קופות כקישור ישיר (כבר קיים) |
