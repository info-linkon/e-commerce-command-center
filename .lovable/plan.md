## הבעיה

הזמנה 253 סומנה באתר כ"איסוף עצמי" (אין משלוח, `shipping_cost = 0`, `notes` מתחיל ב־`🏪 איסוף עצמי`, `shipping_city = "איסוף עצמי"`). למרות זאת, ב־CRM מסך ההזמנה דורש לבצע **שיוך משלוח** לחברת משלוחים לפני שניתן לרשום תשלום או לסגור את ההזמנה.

### שורש הבעיה

ב־`src/pages/orders/OrderDetail.tsx` (שורה 549) הקומפוננטה `DeliveryAssignment` מוצגת עבור **כל הזמנה** שהוקצתה ולא בוטלה — בלי להבדיל בין הזמנת איסוף עצמי לבין הזמנת משלוח.

ב־`PaymentSection.tsx` שורה 306, כפתור "רשום תשלום" מושבת ע״י `disabled={!isDelivered && !hasPayments}`, ו־`isDelivered` מגיע מ־`delivery?.status === "delivered"`. בהזמנת איסוף עצמי לא קיימת רשומת `deliveries`, ולכן `isDelivered=false` תמיד → המשתמש נאלץ לפתוח משלוח מזויף לחברת שליחויות רק כדי לרשום שהלקוח שילם וקיבל את ההזמנה.

## הפתרון

זיהוי הזמנת איסוף עצמי לפי הטביעה הקיימת:
```ts
const isPickup =
  order.shipping_city === "איסוף עצמי" ||
  (order.notes || "").startsWith("🏪 איסוף עצמי");
```
(נשמור את הזיהוי כעוזר ב־helper בתוך `OrderDetail` כדי לא לחזור עליו.)

### 1. `OrderDetail.tsx`

- **כשההזמנה היא איסוף עצמי**: לא להציג כלל את `<DeliveryAssignment>`. במקומה להציג כרטיס סטטוס קצר ("🏪 איסוף עצמי — ממתין לאיסוף בחנות / נאסף") עם כפתור "סמן כנמסר ללקוח" שמתפקד אנלוגית ל־"סמן נמסר" של משלוח רגיל.
- להעביר ל־`PaymentSection` `isDelivered={isPickup ? pickedUp : delivery?.status === "delivered"}` כאשר `pickedUp` הוא דגל המתאר שהאיסוף בוצע.

**שאלת מימוש**: היכן לשמור את עובדת ה"נאסף"? שתי אופציות —
- **א.** לאפשר רישום תשלום בהזמנת איסוף עצמי **בלי** דרישת "נמסר" כלל (פשוט יותר; הפעולה הפיזית של איסוף נעשית בחנות בזמן התשלום).
- **ב.** ליצור עדיין רשומת `deliveries` עם `delivery_company` פנימית בשם "איסוף עצמי" (אוטומטית כשמסמנים "נאסף"), כדי לשמר את אותו צינור סטטוסים והיסטוריה.

אני ממליץ על **אופציה א** — היא תואמת לרוח של "איסוף עצמי" (אין שום שלב משלוח), פותרת את הבעיה ללא יצירת ישויות מזויפות, ולא דורשת מיגרציה. הסטטוס "הושלמה" של ההזמנה כבר מסמן שהלקוח קיבל ושולם.

### 2. `PaymentSection.tsx`

הוספת prop חדש `isPickup: boolean`. שינוי שורה 306 ושורה 472 כך שעבור איסוף עצמי לא תידרש מסירה:
```tsx
disabled={!isPickup && !isDelivered && !hasPayments}
```
זה משחרר את כפתור "רשום תשלום" ואת ההצעה לשלוח לינק תשלום עבור הזמנות איסוף עצמי.

### 3. תצוגה עדינה יותר

ב־`OrderDetail.tsx` להוסיף שורה בכרטיס הראשי המבליטה "🏪 איסוף עצמי" (Badge) ליד מספר ההזמנה / כתובת המשלוח, כדי שמיד כשנכנסים להזמנה רואים את אופי ההגעה ולא רק בעמודת הערות.

### 4. בדיקה רגרסיבית

- הזמנת משלוח רגילה → קומפוננטת `DeliveryAssignment` ממשיכה לעבוד כמקודם, חובה לשייך חברת משלוחים לפני תשלום.
- הזמנת POS עם איסוף → אותה לוגיקה (POS כבר מסמן `pickup` ב־`deliveryMethod`; לוודא שגם ההזמנה שנוצרת ממנו מקבלת `shipping_city = "איסוף עצמי"` כדי שהזיהוי יעבוד גם שם).
