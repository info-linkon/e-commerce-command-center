# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

- `npm run dev` — Vite dev server on port 8080 (HMR overlay disabled).
- `npm run build` — Production build to `dist/`. `npm run build:dev` builds in development mode (used by the Lovable preview).
- `npm run lint` — ESLint over the repo (`@typescript-eslint/no-unused-vars` is turned off; `react-refresh/only-export-components` is a warn).
- `npm run test` — Run Vitest once (jsdom). `npm run test:watch` for watch mode.
- Single test: `npx vitest run src/path/to/file.test.ts`; by name: `npx vitest run -t "substring"`. Test glob is `src/**/*.{test,spec}.{ts,tsx}` with setup in `src/test/setup.ts`.
- Bun lockfile (`bun.lock`, `bun.lockb`) is committed alongside `package-lock.json`; most contributors use npm (matching `deploy.yml`). Don't regenerate the other lockfile casually.

## Architecture

This is a Lovable.dev project (`lovable-tagger` runs as a Vite plugin in dev; edits made on lovable.dev commit back to this repo). It's a single Vite/React SPA that serves **two apps from one bundle**, wired up in `src/App.tsx`:

1. **Public storefront** at `/` — wrapped in `WebLayout` (`src/components/web/WebLayout.tsx`). RTL, Arabic-primary with Hebrew toggle via `LanguageProvider` / `useLanguage` (the `t(ar, he)` helper is used everywhere for bilingual strings; default locale is `ar`, persisted to `localStorage`). Meta Pixel is initialised here.
2. **CRM admin** at `/crm/*` — wrapped in `ProtectedRoute` + `AppLayout` with `AppSidebar`. Requires a Supabase auth session; login lives at `/crm/auth`.

Extra public entry points outside both layouts: `/inv/:code` (invoice shortlink → `InvoiceRedirect`) and `/pay/:orderNumber` (payment shortlink → `PaymentRedirect`). Keep the `*` catch-all last when adding routes.

### Backend: Supabase

- Client is in `src/integrations/supabase/client.ts` with the project URL and anon key hardcoded (the file is auto-generated by Lovable — don't edit it manually; prefer env consumption elsewhere). The generated `Database` type in `src/integrations/supabase/types.ts` is the source of truth for table shapes.
- Migrations are timestamped SQL files in `supabase/migrations/`. Edge Functions live under `supabase/functions/<name>/index.ts`. `supabase/config.toml` lists functions with `verify_jwt = false` (public webhooks / redirects) — anything invoked from the public website or third-party callback goes here (WooCommerce sync/webhooks, HYP payments, SMS, Ezcount docs, `pay-redirect`, `order-summary`, `meta-product-feed`, etc.).
- Auth: `src/hooks/useAuth.tsx` provides `AuthProvider` and wraps `supabase.auth.onAuthStateChange` + `getSession`. `ProtectedRoute` redirects to `/crm/auth` when there's no session.

### Data layer pattern

- **One hook file per domain** in `src/hooks/` (e.g. `useOrders.ts`, `useProducts.ts`, `useInventory.ts`, `useBundles.ts`, `useCoupons.ts`). Each exports a set of `useQuery`/`useMutation` hooks built on `@tanstack/react-query` and the Supabase client. When adding data access, extend the matching domain hook instead of calling `supabase` directly from components.
- Mutations typically call an edge function for side effects (e.g. order status changes invoke `woo-sync` via `supabase.functions.invoke("woo-sync", ...)` and track a `woo_sync_status` column on the row — see `syncOrderStatusToWoo` in `useOrders.ts`).
- Toasts use `sonner` (imported as `toast`). There's also a legacy `useToast` hook wired to the shadcn `Toaster`.
- Client-only state: Zustand (`src/lib/web-cart-store.ts` for the storefront cart).

### Third-party integrations (important)

- **WooCommerce** — product/stock/order sync via `supabase/functions/woo-sync`, `woo-stock-update`, `woo-product-sync`, `woo-webhook`; helpers in `src/lib/wooStockSync.ts` and `src/lib/wooProductSync.ts`. Admin UI at `/crm/woo-sync`.
- **HYP** payment gateway — `hyp-create-payment`, `hyp-payment-link`, `hyp-verify-payment` functions; settings at `/crm/admin/hyp-settings`.
- **Ezcount** invoicing — `ezcount-doc` function.
- **InfoRu** SMS — `send-sms`, `send-otp`, `order-sms-trigger`.
- **Meta Pixel** — `src/lib/meta-pixel.ts` + settings at `/crm/admin/meta-pixel`; product feed served by `meta-product-feed`.

### UI / styling

- shadcn-ui components in `src/components/ui/` (generator config in `components.json`; base color `slate`, alias `@/components`).
- Tailwind with custom tokens (`sand`, `desert`, `gold`, `warm` plus the usual shadcn CSS variables) and the `Cairo` font family — see `tailwind.config.ts`. Custom animations: `fade-in`, `slide-in-right`.
- Path alias: `@/*` → `src/*` (configured in `vite.config.ts`, `vitest.config.ts`, and `tsconfig*.json`). Use it consistently instead of relative imports.
- Layouts are RTL (`dir="rtl"`); the CRM is Hebrew-only, the public site is bilingual.

### Deployment

- `.github/workflows/deploy.yml` runs on push to `main`: `npm install` → `npm run build` → FTP-deploy `dist/` to cPanel (uses `dangerous-clean-slate: true`, which wipes the remote dir). Keep the workflow branch filter in mind when opening PRs.
